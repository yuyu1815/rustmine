Purpose: document the Protocol 775 jar-backed proof package for Configuration
serverbound `minecraft:select_known_packs` framed dispatch/decode without
expanding it into registry hydration, known-pack negotiation completion, or Play
readiness.

## Proves

The official jar emits the full Configuration serverbound
`minecraft:select_known_packs` frame for `ServerboundSelectKnownPacks`,
dispatches that frame back to `ServerboundSelectKnownPacks`, preserves the
official known-pack list generated by `KnownPack.vanilla(String)`, and leaves no
unread bytes after official decode.

The generated answer, not this note, owns the exact packet id, frame bytes,
body bytes, known-pack namespace/id/version values, and list encoding.

## Current Rust Result

The manifest-declared Rust oracle test reads the generated answer and calls
Stevenarella `packet::packet_by_id` with the official packet id and body. In
the current checkout, that test passes: Protocol 775 Configuration serverbound
id `0x07` dispatches to the select-known-packs body decoder and consumes the
official known-pack list body bytes.

## Does Not Prove

This does not prove registry hydration, known-pack negotiation completion,
Configuration completion, Play entry, world hydration, or any later
client-load phase.
